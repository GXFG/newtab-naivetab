# ❓ FAQ

## 🐛 遇到不明错误

可先尝试以下操作：

1. **清除缓存**：`通用设置` → `清除缓存`，重置后刷新页面再次检查
2. **重置设置**：`通用设置` → `重置设置`，可选择 **快速重置**（保留用户数据）或 **完全重置**（回到默认值）
3. **检查浏览器版本**：确保浏览器版本较新，过旧的浏览器可能不支持部分功能

如果以上方法无法解决问题，欢迎通过以下渠道反馈：

- [GitHub Issue](https://github.com/GXFG/newtab-naivetab/issues)
- [Email](mailto:gxfgim@outlook.com?subject=NaiveTab+Feedback)

## 🔑 快捷键不生效

**指令键盘 / 键盘书签的快捷键没有反应？**

1. 确认修饰键设置正确，指令键盘和键盘书签建议使用不同的修饰键组合
2. 检查是否在浏览器特殊页面（`chrome://`、`edge://`、`about://` 等），这些页面无法触发快捷键
3. 检查是否命中了域名黑名单，当前网站是否在黑名单中
4. 确认没有与其他扩展的快捷键冲突（浏览器设置 → 扩展 → 快捷键 中可查看冲突）

**输入框中意外触发了快捷键？**

- 默认情况下输入框中不会触发指令，如需在输入框中也触发，可在设置中开启「输入框中触发」选项

## 🔒 权限相关

**安装后提示需要权限？**

NaiveTab 采用可选权限机制，核心功能无需额外权限。以下功能需要单独授权：

| 功能 | 权限 | 触发时机 |
|------|------|----------|
| 书签文件夹 | `bookmarks` | 首次拖入书签文件夹组件时 |
| 天气通知 | `notifications` | 首次有天气预警需要推送时 |

**授权弹窗不出现？**

- 确保你主动使用了相关功能（如拖入书签文件夹组件），权限仅在功能使用时才会请求
- 如果之前拒绝了权限，可在浏览器设置中手动开启

## ☁️ 同步功能相关

- 设置同步依赖浏览器的账号同步功能，需要先登录浏览器账号
- 如果同步失败，可尝试导出设置到本地进行备份
- 部分配置项（如键盘书签）的存储接近浏览器同步上限时，建议减少书签数量

**多设备配置不一致？**

- 确保两台设备都登录了同一个浏览器账号
- 同步可能有几秒到几分钟的延迟，等待片刻后刷新
- 如长时间未同步，尝试在一台设备上导出 → 另一台设备上导入

## 🌤 天气组件无法使用

- 天气组件依赖和风天气 API，需确保网络连接正常
- 如果天气数据加载失败，可尝试在设置中刷新或更换城市
- 内置 API Key 有调用频率限制，大量使用时建议申请自己的 Key

## 🎨 自定义样式不生效

- 修改设置后，样式应该立即更新
- 如果没有生效，尝试刷新页面（`Ctrl+R` / `Cmd+R`）
- 检查是否在深色/浅色主题下分别设置了颜色值，两种主题的配置是独立的

## 📁 书签文件夹显示为空

- 确认浏览器中确实有书签
- 书签文件夹组件默认展示根目录，点击文件夹可进入子目录
- 首次使用需要授权书签权限，如果之前拒绝授权，可在浏览器设置中重新允许

## 🖼️ 背景图不显示

- 本地上传的图片存储在 IndexedDB 中，清除浏览器数据会导致丢失
- Bing 每日一图需要网络连接才能获取
- 如果图片加载缓慢，可能是图片文件过大，建议上传压缩后的图片
- 幻彩背景为纯 CSS 渲染，无需网络，切换来源为「幻彩」即可

## 🎨 自动取色不生效

- 确保在设置面板中开启了「自动主题色」/「自动背景色」开关
- 自动取色在切换壁纸或幻彩配色后立即生效
- 不同外观模式（浅色/深色）独立取色，切换外观后自动对应更新

## 📊 性能相关

**新标签页加载速度慢？**

- NaiveTab 首屏使用小图快速加载，大图异步加载
- 如果上传了大量本地背景图，首屏恢复可能稍慢
- 组件数量过多也可能影响性能，建议使用专注模式隐藏不需要的组件

**浏览器内存占用高？**

- NaiveTab 仅在新标签页运行时占用少量内存
- 如果内存持续增高，尝试刷新新标签页（`Ctrl+R` / `Cmd+R`）

## 📦 安装相关

**在 Chrome 应用商店安装失败？**

- 确保使用的是 Chrome 或基于 Chromium 的浏览器（Edge、Arc 等）
- 网络环境可能需要代理才能访问 Chrome Web Store
